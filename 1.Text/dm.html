Homework: Editing text with markdown and pandoc

  1. Go to the essay you contributed – the one you pushed to the Github repository. Open that up in Sublime Text.

    Let’s fix the headings. If Pandoc recognized the headings, make sure they’re at the correct level (first-level heads? second-level heads?). If it didn’t recognize your headings, add them.

    Did Pandoc recognize block (indented) quotations? If not, mark those down too.

    Lists? Images? Anything else?

    Once you’ve done that, commit and push your changes back to the Github repo.

  2. Have a look at the “OYO-PandocTutorial” file, and try out the examples, using your essay.

    This is work you should probably do outside of the Github folder, because you probably don’t want to contribute all the things you create while working with it. So cd someplace outside that folder, and make some new files. And don’t push them back to the repo, thanks.

  3. I’ve included here a sample CSS stylesheet for you to play with. Make a copy of this file (just “Save as” in Sublime Text) We’ll talk about CSS a lot more over the next few sessions, but you can use this one as an example when you convert to HTML. It is made terrifically ugly on purpose: to provoke you into making changes!

  4. Try doing some global search-and-replace on your Essay. Can you change all of your headings to another level? E.g., make all your first-level headings into second-second level headings, or vice-versa.

    And, have a look at “OYO-RegExTutorial” – there are some challenge exercises at the end of that file.
